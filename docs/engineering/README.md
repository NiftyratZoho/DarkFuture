# Engineering Docs

These documents define the shared contract for agents working on the faithful, turn-based Dark Future adaptation.

Agents must read these before producing code or data files:

- `architecture.md` - engine boundaries and core module layout.
- `agent-coding-guidelines.md` - coding rules, ownership boundaries, and output expectations.
- `agent-workflow.md` - agent roles, review gates, and section handoff format.
- `rules-data-schema.md` - canonical data conventions for rules, track geometry, dice, and logs.
- `testing-strategy.md` - required validation approach for extracted rules and engine behaviour.

The current priority is still extraction and proofreading. These documents exist so that, once rules sections are clean, implementation agents produce compatible work.
